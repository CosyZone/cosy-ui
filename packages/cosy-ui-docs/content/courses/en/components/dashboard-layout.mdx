## Usage Examples

### Basic Configuration

```astro
---
import { DashboardLayout } from '@coffic/cosy-ui';

// Define navigation items
const navItems = [
  { href: "/dashboard", icon: "home", text: "Dashboard" },
  { href: "/dashboard/users", icon: "users", text: "User Management", badge: "5" },
  { 
    href: "/dashboard/content", 
    icon: "document", 
    text: "Content Management",
    items: [
      { href: "/dashboard/content/articles", icon: "document", text: "Article Management" },
      { href: "/dashboard/content/categories", icon: "star", text: "Category Management" }
    ]
  },
  { href: "/dashboard/settings", icon: "settings", text: "System Settings" }
];
---

<DashboardLayout
  title="Admin Dashboard"
  navItems={navItems}
  userName="Administrator"
  systemName="Content Management System"
>
  <h1>Welcome to Admin Dashboard</h1>
  <p>This is the main content area</p>
</DashboardLayout>
```

## Features

### Responsive Design

- **Desktop**: Shows complete sidebar and all features
- **Tablet**: Collapsible sidebar with good interaction experience
- **Mobile**: Sidebar converts to drawer mode to save screen space

### Sidebar Size Configuration

- **SM (192px)**: Compact layout, mobile-friendly, suitable for simple navigation
- **MD (256px)**: Default size, balanced choice, suitable for most scenarios
- **LG (288px)**: Spacious layout, suitable for displaying more content
- **XL (320px)**: Widest layout, suitable for complex multi-level navigation structures

### Sidebar Theme System

DashboardLayout provides 10 preset themes based on daisyUI's semantic color system:

- **default**: Default light gray, the best choice for general scenarios
- **dark**: Dark theme with modern design, reduces visual fatigue
- **neutral**: Neutral tones for professional business scenarios
- **primary**: Brand primary color, emphasizes corporate identity
- **secondary**: Secondary color with gentle visual effects
- **accent**: Accent color for highlighting key features
- **info**: Information blue, suitable for data display interfaces
- **success**: Success green for health, finance, and other positive applications
- **warning**: Warning yellow for monitoring and alert systems
- **error**: Error red for security, auditing, and other serious applications

All themes automatically handle text contrast to ensure good readability and accessibility compliance.

### Content Theme System

DashboardLayout provides 11 content area themes for flexible configuration of main content background styles:

- **card**: Default card style with clear boundaries and subtle shadow effects
- **transparent**: Transparent background, removes container restrictions, provides maximum design freedom
- **base**: Basic background color using base-200 tone, gentle visual effects
- **neutral**: Neutral tones for professional business feel
- **primary**: Brand primary color, enhances corporate identity
- **secondary**: Secondary color with gentle brand representation
- **accent**: Accent color for highlighting important content
- **info**: Information blue, suitable for data display and analytics interfaces
- **success**: Success green for health management, financial growth, and other positive applications
- **warning**: Warning yellow for monitoring systems, pending states, and scenarios requiring attention
- **error**: Error red for security auditing, error handling, and other serious applications

The content theme system is particularly suitable for:
- **Admin Dashboards**: card theme provides professional, clear management interfaces
- **Creative Tools**: transparent theme offers unlimited design space
- **Brand Showcase**: primary/secondary/accent themes enhance brand identity
- **Data Dashboards**: info theme creates professional data analytics atmosphere
- **Status Monitoring**: warning/error themes highlight system status and issues

### State Management

- Automatically saves sidebar collapse state to local storage
- Restores user preferences after page refresh
- Supports programmatic control of sidebar state

### Navigation System

- Supports multi-level navigation structure
- Automatically highlights current page navigation item
- Supports badge display (like message count, pending tasks, etc.)

## Best Practices

### Navigation Structure Design

1. **Keep it Simple** - Recommend no more than 7 main navigation items
2. **Logical Grouping** - Group related features into sub-menus
3. **Priority Ordering** - Place frequently used features at the top
4. **Consistent Icons** - Use semantic icons to improve recognition

### Content Area Layout

1. **Card Wrapping** - Use `card` component to wrap content areas
2. **Consistent Spacing** - Use consistent padding and margins
3. **Responsive Considerations** - Ensure content readability across all screen sizes

### Performance Optimization

1. **Lazy Loading** - Consider lazy loading for large pages
2. **State Caching** - Utilize local storage appropriately
3. **Component Reuse** - Extract common components to avoid duplication

## Style Customization

The DashboardLayout component follows cosy-ui styling conventions:

- Uses daisyUI's `drawer`, `navbar`, `menu`, `card` components
- Tailwind CSS classes require `cosy:` prefix (e.g., `cosy:bg-base-200`)
- Supports complete responsive design
- Built-in light and dark theme support
