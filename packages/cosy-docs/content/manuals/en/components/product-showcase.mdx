## Introduction

The ProductShowcase component is a container component for ProductShowcaseItem, designed to display detailed information about multiple products. It features an Apple-style large image display design with alternating left-right layouts, providing an elegant product showcase experience. Perfect for product showcase pages, product detail pages, and similar scenarios.

Component Features:

- Large Image Display - Highlights product images with high-quality large image presentation
- Alternating Layout - Automatic left-right alternating layout for enhanced visual hierarchy
- Information Hierarchy - Clear information structure: labels, titles, descriptions, prices, buttons
- Apple Style - Adopts Apple's clean and elegant design philosophy
- Responsive Design - Excellent display across different devices

## Design Philosophy

The ProductShowcase component follows Apple's design philosophy:

1. **Large Image Display** - Highlights product images for intuitive user understanding
2. **Alternating Layout** - Left-right alternating layout enhances visual hierarchy and avoids monotony
3. **Information Hierarchy** - Clear information structure: new product label → product title → description → price → action buttons
4. **Clean and Elegant** - Minimalist design that highlights the product itself
5. **Responsive** - Provides excellent user experience across different devices

## Best Practices

1. **Image Quality** - Use high-quality product images, recommended size 800x600 or higher
2. **Concise Descriptions** - Product descriptions should be concise and highlight key selling points
3. **Clear Pricing** - Price display should be clear and prominent with appropriate font size
4. **Button Copy** - Button text should clearly express the intended action
5. **New Product Labels** - Use new product labels appropriately to highlight new products

## Notes

- The component automatically applies alternating left-right layout to each product item
- Background color is applied to all product items; use ProductShowcaseItem for individual settings
- Images should be high resolution to ensure good display in large image showcase
- The component uses Container internally to ensure content doesn't become too wide
