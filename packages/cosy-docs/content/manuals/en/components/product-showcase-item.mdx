## Introduction

The ProductShowcaseItem component is designed to display detailed information about a single product using an Apple-style large image display design. It supports alternating left-right layouts, new product labels, price display, and action buttons. Perfect for product detail pages, product showcase pages, and similar scenarios.

Component Features:

- Large Image Display - Highlights product images with high-quality large image presentation
- Alternating Layout - Supports left-right alternating layout for enhanced visual hierarchy
- Information Hierarchy - Clear information structure: labels, titles, descriptions, prices, buttons
- Apple Style - Adopts Apple's clean and elegant design philosophy
- Responsive Design - Excellent display across different devices

## Design Philosophy

The ProductShowcaseItem component follows Apple's design philosophy:

1. **Large Image Display** - Highlights product images for intuitive user understanding
2. **Information Hierarchy** - Clear information structure: new product label → product title → description → price → action buttons
3. **Clean and Elegant** - Minimalist design that highlights the product itself
4. **User-Friendly Interaction** - Provides clear action buttons to guide user behavior
5. **Visual Balance** - Achieves visual balance through left-right layout

## Best Practices

1. **Image Quality** - Use high-quality product images, recommended size 800x600 or higher
2. **Concise Descriptions** - Product descriptions should be concise and highlight key selling points
3. **Clear Pricing** - Price display should be clear and prominent with appropriate font size
4. **Button Copy** - Button text should clearly express the intended action
5. **New Product Labels** - Use new product labels appropriately to highlight new products

## Layout Description

The component uses a two-column grid layout:

- **Left Column**: Product information (label, title, description, price, buttons)
- **Right Column**: Product image

When `align="right"`, the layout is reversed:
- **Left Column**: Product image
- **Right Column**: Product information

## Notes

- Images should be high resolution to ensure good display in large image showcase
- The component uses Container internally to ensure content doesn't become too wide
- Background color is applied to the entire component area
- Button click events need to be handled by yourself; the component only provides callback interfaces
