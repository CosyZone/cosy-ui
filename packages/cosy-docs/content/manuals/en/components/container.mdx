## Introduction

The Container component is a basic layout container used to limit content width and center display. It provides various sizes and padding options, suitable for different layout needs.

## Props

### aspectRatio
Aspect ratio (width/height). When set, the container will maintain this ratio, such as 16/9, 4/3, 1, etc. When aspectRatio is set, the container will always maintain this ratio regardless of screen size changes.
<ContainerPackage.Containers.AspectRatio />

### background
Background color type, supporting background colors and opacity variants, such as: base-100, primary, secondary, accent, neutral, info, success, warning, error, primary/10, secondary/20, etc.
<ContainerPackage.Containers.Background />

### backgroundImage
Background image source, supports local ImageMetadata and remote URLs. When provided, it renders as a full-cover background layer. It can be combined with `background` color for better contrast.
<ContainerPackage.Containers.BackgroundImage />

### border
Border size. Optional values: none, sm, md, lg, xl.
<ContainerPackage.Containers.Border />

### borderColor
Border color, supports all predefined colors and opacity variants, such as: primary, success, primary/50, etc.
<ContainerPackage.Containers.BorderColor />

### centered
Whether to center the display. When set to false, the container will be left-aligned.
<ContainerPackage.Containers.Centered />

### class
Custom CSS class name for adding additional styles.
<ContainerPackage.Containers.CustomClass />

### contentBorder

<Alert type="info" variant="soft" closable={false}>
  Only effective when `fit` is `contain` or `cover`.
</Alert>

Whether to add a border to the proportional content area (the inner ratio box in fit mode).
<ContainerPackage.Containers.ContentBorder />

### fit
Content fitting mode. Options: `none` (default), `contain` (preserve ratio, fill as much as possible without overflow), `cover` (preserve ratio, fill and possibly crop). Must be used together with `aspectRatio`, and it is recommended that the parent container has an explicit height (e.g., `height="lg"`).
<ContainerPackage.Containers.Fit />

### flex
Flex layout direction. Optional values: row, col, row-reverse, col-reverse. If not set, flex layout is not enabled.
<ContainerPackage.Containers.Flex />

### gap
Flex item spacing. Optional values: none, xs, sm, md, lg, xl. Only effective when flex layout is enabled.
<ContainerPackage.Containers.Gap />

### height
Container height. Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl, 6xl, screen, auto. none means no height is set.
<ContainerPackage.Containers.Height />

### items
Flex item horizontal alignment. Optional values: start, end, center, baseline, stretch. Only effective when flex layout is enabled.
<ContainerPackage.Containers.Items />

### justify
Flex item vertical alignment. Optional values: start, end, center, between, around, evenly. Only effective when flex layout is enabled.
<ContainerPackage.Containers.Justify />

### margin
Outer margin size. Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl, 6xl.
<ContainerPackage.Containers.Margin />

### padding
Inner padding size. Optional values: none, sm, md, lg, xl.
<ContainerPackage.Containers.Padding />

### py
Vertical padding (top and bottom). Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl.
<ContainerPackage.Containers.PaddingAxis />

### pt
Top padding. Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl.
<ContainerPackage.Containers.PaddingTop />

### pb
Bottom padding. Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl.
<ContainerPackage.Containers.PaddingBottom />

### px
Horizontal padding (left and right). Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl.
<ContainerPackage.Containers.PaddingX />

### pl
Left padding. Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl.
<ContainerPackage.Containers.PaddingLeft />

### pr
Right padding. Optional values: none, xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 5xl.
<ContainerPackage.Containers.PaddingRight />

⚠️ **Note**:
- Cannot use `px` and `pl`/`pr` properties at the same time, as they both set horizontal padding and will cause conflicts
- Cannot use `py` and `pt`/`pb` properties at the same time, as they both set vertical padding and will cause conflicts
- If conflicting properties are provided simultaneously, the component will display an error message.

### rounded
Border radius size. Optional values: none, sm, md, lg, xl, full.
<ContainerPackage.Containers.Rounded />

### width
Container width. Optional values: xs, sm, md, lg, xl, full. Controls the maximum width of the container.
<ContainerPackage.Containers.Widths />
